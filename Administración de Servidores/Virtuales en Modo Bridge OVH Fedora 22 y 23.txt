Configurar los hosts para las distribuciones de 
virtualización que permiten el modo bridge
-----------------------------------------------


Introducción
------------
OVH permite configurar las máquinas virtuales en modo bridge. 
Para que funcione la configuración de red, es necesario realizar 
algunos cambios en las máquinas virtuales.


Requistos previos
-----------------
Es necesario disponer de un servidor dedicado con VMware ESXi, 
Citrix Xen Server o Hyper-V, haber asignado una MAC virtual a 
una IP failover y saber conectarse por SSH.


Determinar la puerta de enlace del host
---------------------------------------
Para poder configurar las máquinas virtuales, es necesario 
conocer la puerta de enlace de la máquina host (nsxxx.ovh.net, ksxxx.ovh.net...). Para ello, debe sustituir el último grupo 
de cifras de la IP de su máquina por 254. Si no conoce dicha 
IP, la encontrará en el email de instalación de la máquina.

Por ejemplo:

IP del servidor (host): 123.456.789.012
IP de la puerta de enlace del host (igual que la IP, pero terminada en .254):
123.456.789.254

Si utiliza una puerta de enlace incorrecta, recibirá 
alertas y podría bloquear la IP afectada.


Determinar la máscara de red
----------------------------
Si utiliza una IP failover normal o un bloque RIPE, 
debe utilizar la máscara de red por defecto para la 
configuración de bridge:

Netmask: 255.255.255.255

Si utiliza una máscara de red incorrecta, experimentará 
problemas de conexión.


Fedora
------
Importante: La puerta de enlace de la máquina virtual nunca debe ser 
la IP del servidor dedicado ni la IP failover, sino la puerta de 
enlace asignada al servidor dedicado. En ningún caso se debe utilizar 
route add default gw dev eth0. De lo contrario, podría bloquearse la 
IP para el servidor virtual.

Para determinar la puerta de enlace que debe utilizar, sustituya el 
último grupo de cifras de la IP principal del servidor dedicado por 254. 
Por ejemplo:

IP failover: YYY.YYY.YYY.YYY
IP principal del servidor dedicado: XXX.XXX.XXX.XXX
IP de la puerta de enlace para la máquina virtual: XXX.XXX.XXX.254

En adelante, llamaremos a esta puerta de enlace GATEWAY_VM.


Contenido del archivo: /etc/sysconfig/network-scripts/ifcfg-eth0

DEVICE=eth0
BOOTPROTO=none
ONBOOT=yes
USERCTL=no
IPV6INIT=no
PEERDNS=yes
TYPE=Ethernet
NETMASK=255.255.255.255
IPADDR=IP.FAILOVER
GATEWAY=GATEWAY_VM
ARP=yes
HWADDR=MAC:VIRTUAL


Contenido del archivo: /etc/sysconfig/network-scripts/route-eth0

GATEWAY_VM dev eth0
default via GATEWAY_VM dev eth0


Contenido del archivo: /etc/resolv.conf

nameserver 213.186.33.99
